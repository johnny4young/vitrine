# Support

## Usage questions

Open a [question or support request](https://github.com/johnny4young/vitrine/issues/new?template=support_request.yml)
when you need help using Vitrine or building it from source. Include the Vitrine
version, macOS version, installation method, what you tried, and any privacy-safe
diagnostics that help reproduce the problem.

## Bugs and feature requests

- Use the [bug report](https://github.com/johnny4young/vitrine/issues/new?template=bug_report.yml)
  for reproducible incorrect behavior.
- Use the [feature request](https://github.com/johnny4young/vitrine/issues/new?template=feature_request.yml)
  for product ideas that preserve Vitrine's local-first privacy boundary.

Search existing issues before opening a new one. Keep one problem or proposal per
issue so it can be triaged and closed independently.

## Security and sensitive reports

Do not post vulnerabilities, credentials, private code, or personal information in
an issue. Follow [SECURITY.md](SECURITY.md) and use GitHub's private vulnerability
reporting flow instead.

Possible Code of Conduct violations must also be reported privately as described in
[CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
